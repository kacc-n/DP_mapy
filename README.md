# Eye-Tracking Map Control

This project is part of a diploma thesis titled **"Development and testing of gaze controlled map interface"** developed at the **[Department of Geoinformatics](https://www.geoinformatics.upol.cz/), PalackГЅ University, Olomouc**.

It is a web-based application for user testing of different map control approaches combining gaze-based and mouse-based interaction. The application uses gaze data transmitted via a WebSocket server and visualizes map data using the MapLibre GL JS library.

---

## Key Features

- **Gaze Panning**: Looking outside the screen edges moves the map in that direction
- **Gaze Zooming**: A two-step zoom system designed to prevent the "Midas touch" problem
    1. Fixate gaze on the center of the screen for 1 second
    2. Look **up** to Zoom In or **down** to Zoom Out
- **Mouse Panning & Zooming**: Standard scroll and drag interaction (used in passive-movement variants)
- **Legend Interaction**: Layer highlighting via either gaze dwell (1 second) or mouse click depending on the map variant
- **Restaurant Labels**: Text labels showing name and seating capacity, visible only when the restaurant layer is selected
- **Visual Feedback**: Real-time gaze dot and active edge zone highlighting
- **AprilTag Loading**: Dynamic loading and positioning of calibration markers based on `surface_layout.yaml` generated by GazedeckConsole

## Getting Started

This application runs in a web browser but requires a specific hardware and software setup to function correctly.

### Prerequisites

- **Hardware**: [Pupil Labs](https://pupil-labs.com/) eye-tracking glasses (e.g. Pupil Labs Neon)
- **Software**: A running instance of [**GazeDeck Console**](https://github.com/misavojte/GazeDeck) developed by Mgr. Michaela VojtД›chovskГЎ
- **Network**: A stable Wi-Fi or mobile hotspot without restrictive firewalls (university networks may block WebSocket traffic)

### Running the Application

1. **Prepare the environment**
   - Connect your eye-tracking glasses
   - Launch GazeDeck Console on a computer

2. **Launch the application**
   - Open a web browser on the same computer
   - Navigate to the project GitHub Pages URL: [GitHub Pages link](PASTE_YOUR_URL_HERE)
   - Or if running locally, open `index.html` (the root one, not inside `maps/`)

## Controls

### Gaze controls (used in Active variants)

| Action | Gaze Gesture |
|--------|--------------|
| **Pan map** | Look outside any screen edge (top, bottom, left, right, or corners). The edge highlights and the map moves in that direction. |
| **Activate zoom** | Fixate gaze on the center of the screen for 1 second until the indicator disappears. |
| **Zoom in** | After zoom activation, look at the **upper** half of the screen. |
| **Zoom out** | After zoom activation, look at the **lower** half of the screen. |
| **Select legend layer** | Look at a legend tile for 1 second until the dwell bar fills. |

### Mouse controls (used in Passive variants)

| Action | Input |
|--------|-------|
| **Pan map** | Click and drag |
| **Zoom** | Scroll wheel |
| **Select legend layer** | Click on a legend tile |

### Keyboard shortcuts (all variants)

| Key | Action |
|-----|--------|
| `H` | Fly back to the starting position |
| `Space` | Advance to the next slide in the testing session |

## Troubleshooting 

**Map not responding to gaze:**
- Check the browser console (F12) for `вњ… GazeDeck connected!`. If you see `вќЊ GazeDeck connection error`, make sure GazeDeck Console is running.
- Make sure you are not on a restrictive network that blocks WebSocket connections.

**Spacebar not advancing to the next map:**
- This can happen if the user has clicked inside the map iframe. Each `map.js` sends a `postMessage` to the parent window to handle this вЂ” make sure you are running the root `index.html` and not opening individual maps directly.
