# Eye-tracking Map Control

This project is part of a diploma thesis titled **"Development and testing of gaze controlled map interface"** developed at the **[Department of geoinformatics](https://www.geoinformatics.upol.cz/), Palacký University, Olomouc**.

It is a web-based application that enables hands-free map navigation (panning and zooming) using only the user's gaze. The application utilizes gaze data transmitted via a WebSocket server and visualizes map data using the known Leaflet library.


## Key Features

- **Panning**: Looking at outside of the screen activates map movement in that direction
- **Zooming**: A two-step zoom system designed to prevent the "Midas touch" problem
    1. Fixate gaze on the center of the screen
    2. Look **up** to Zoom In or **down** to Zoom Out
- **Visual Feedback**: Real-time gaze visualization (gaze dot) and active zone highlighting
- **Automatic AprilTags Loading**: Dynamic loading and positioning of markers (AprilTags) based on the 'surface_layout.yaml' configuration generated by GazedeckConsole.exe

## Built With

- **HTML5 / CSS3** – UI structure and styling
- **JavaScript (ES6+)** – Core application logic (Vanilla JS)
- **[Leaflet.js](https://leafletjs.com/)** – Interactive map library
- **WebSocket** – Real-time communication with the GazeDeck server
- **js-yaml** – Parsing YAML configuration files for tag layout

## Getting Started

This application runs in a web browser but requires a specific backend setup and network environmnent to function correctly.

### Prerequisites

- **Hardware**: [Pupil Labs](https://pupil-labs.com/) eye-tracking glasses (I used Pupil Labs Neon)
- **Software**: A running instance of [**Gazedeck Console**](https://github.com/misavojte/GazeDeck) developed by Mgr. Michaela Vojtěchovská
- **Network**: A stable Wi-Fi (or mobile hotspot) without restrictive firewalls (f.e. university network may block WebSocket/P2P traffic)

### Installationg and Running

This application is hosted via GitHub Pages, so no installation or repository cloning is requiered.

1. **Prepare the Local Environmnent**
- **Connect Hardware**: Ensure your eye-tracking glasses are connected
- **Start Middleware**: Launch the Gazedeck Console application
2. **Launch the Application**
- Open a web browser on the same computer where Gazedeck Console is running
- Navigate to the project URL: [GitHub pages Link here](linkhere)

## Controls

The interface is designed for completely hands-free interaction.

| Action | Gaze Gesture |
|----------|--------------|
| **Pan Map** | Look outside of any edge of the screen (Top, Bottom, Left, Right or Corners). The edge will highlight, and the map will move in that direction. |
| **Activate Zoom** | Fixate your gaze on the center of the screen. Wait 1 second for the indicator to load and disappear. |
| **Zoom In (+)** | After Zoom activation, look at the **top** half of the screen. |
| **Zoom Out (-)** | After Zoom activation, look at the **bottom** half of the screen. |

Note: To stop panning, simply look back towards the center of the screen (away from the active edge zones).

## Project structure

* `index.html` - Main entry point.
* `map.js` - Core logic: Map initialization, WebSocket handling, gaze processing, and zone detection.
* `tagsloader.js` - Handles the dynamic loading and snapping of AprilTags based on YAML configuration.
* `style.css` - Styling for the map, indicators, and calibration tags.
* `my_screen/` - Contains `surface_layout.yaml` and tag images.

Note: If you decide to generate your own AprilTag layout using GazeDeck, you must follow a strict naming convention for the application to recognize the files: ./my_screen/surface_layout.yaml (inside "my_screen" folder is .yaml configuration and .png of AprilTags)

## Troubleshooting

If the map is not responding:

- check the browser console (F12) for the message: `✅ GazeDeck connected successfully!`. If you see `❌ GazeDeck connection error`, ensure Gazedeck is running correctly
- verify you are not connected to a restrictive network which might block the connection
